## 1. Что умеет бот как информационная площадка

| Тип сообщения | Кто получает | Откуда |
|---------------|--------------|--------|
| Транзакционные (запись, напоминание, отмена) | клиент и/или специалист по событию | сайт автоматически |
| Режимные ответы бота (/start, меню) | тот, кто пишет боту | бот |
| **Рассылки (новости, анонсы)** | сегмент, который выбрал админ | **Админка → Telegram** |

Один человек с двумя ролями получает **одно** сообщение рассылки, не два.

## 3. Как отправить сообщение по категориям (целевой UX)

1. Войти на сайт под админ-аккаунтом (`is_staff` / superuser).
2. Открыть **Админ-панель** → **Telegram**.
3. Нажать **Новая рассылка**.
4. Выбрать аудиторию:

| Сегмент | Кому уйдёт |
|---------|------------|
| Только себе (тест) | ваш Telegram |
| Все уникальные | все известные chat_id по одному разу |
| Только клиенты | у кого есть клиентский TG, без specialist Integration (или минус специалисты - как настроено) |
| Только специалисты | Integration.telegram_chat_id |
| Dual | и клиент, и специалист одновременно |

5. Написать текст. Смотреть **превью** (как увидит пользователь).
6. Нажать **Dry-run** → проверить число получателей.
7. **Отправить** (или запланировать, если функция уже есть).
8. Дождаться статуса job: sent / partial / failed.
9. При ошибках открыть лог доставки (заблокировал бота, 429, неверный chat_id).

## 4. Правила хороших информационных сообщений

Единый стиль (Phase 11):

1. Первая строка - роль/тип: `📢 Новости сервиса` / `📅 Напоминание` / `🆕 Запись`.
2. Коротко: 1 мысль = 1 абзац. До ~500-800 символов для рассылок.
3. Без длинных тире; обычный дефис `-`.
4. Одна главная ссылка или одна кнопка WebApp.
5. Не слать одну и ту же рассылку чаще, чем нужно (люди отпишут / заблокируют).
6. Для dual-пользователя не писать «вам как клиенту и как специалисту» дважды - один ясный текст.

Пример рассылки:

```
📢 Новости «Все клиенты здесь»

Обновили запись в Telegram: теперь сайт открывается прямо в приложении.

Открыть сервис: кнопка ниже
```

Пример транзакционного (клиент):

```
📅 Ваша запись подтверждена

Специалист: Иван П.
Когда: 22.07.2026 15:00
Услуга: Консультация
```

Пример транзакционного (специалист):

```
🆕 Новая запись к вам

Клиент: Вася
Когда: 22.07.2026 15:00
Услуга: Консультация
```

## 5. Что делать, если…

| Проблема | Действие |
|----------|----------|
| Сообщение не пришло вам (test_self) | Проверить, что вы нажали /start у бота; Integration/Social привязаны |
| Ушло «слишком многим» | Смотреть dry-run; проверить сегмент; не повторять send |
| Много failed «bot blocked» | Норма: пользователь заблокировал бота; чистить сегмент позже |
| Telegram 429 | Job сам делает backoff; не запускать вторую рассылку параллельно |
| Dual жалуется на два одинаковых | Проверить, что dedup Phase 3 включён; завести тикет |
| Нужно срочно остановить | В админке Stop job (когда будет) / остановить worker рассылок на сервере |

## 6. Безопасность для админа

- Не передавать доступ staff третьим лицам без RBAC.
- Не вставлять в рассылку персональные данные чужих клиентов пачкой.
- Impersonation («войти как пользователь») - только для поддержки, все действия логируются.
- Пароли пользователей из админки **нельзя** посмотреть - только сброс.
